# Shiftify вЂ” Agent Guide

TypeScript CLI for migrating Shopify production store data (products, collections) to a dev store.

## Commands

```bash
node --version                 # must be 18+  (.node-version = v24.13.0)
npm run export                 # export from PROD_SHOP в†’ data/
npm run import                 # import to DEV_SHOP from data/
npm run export -- --only products --only collections
npm run export -- --skip collections    # inverse of --only; cannot combine with --only
npm run import -- --only products
npm run test                   # vitest unit tests
npm run test:integration       # live integration tests (requires .env.test.integration)
npm run codegen                # regenerate src/gql/ from .graphql files
npm run check                  # biome lint + format (read-only check)
npm run format                 # biome format --write
```

No build step вЂ” `tsx` runs TypeScript directly.

## Key Conventions

- **No `.js` extensions** вЂ” `moduleResolution: "bundler"` in tsconfig; omit extensions on all relative imports
- **Single quotes, no semicolons, 2-space indent, 100-char line width** (Biome)
- **No build step** вЂ” `tsx` executes TypeScript directly; `tsc --noEmit` is type-check only
- **Arrow functions only** вЂ” never use `function` declarations; use `const foo = () =>` everywhere (including exports and async functions)
- **No top-level await** вЂ” async logic lives inside `const main = async () =>`
- **`type` over `interface`** вЂ” always use `type Foo = { ... }`, never `interface Foo { ... }`
- **No type casts** вЂ” `value as SomeType` and `value!` are banned (`noExplicitAny` + `noNonNullAssertion` in Biome); `as const` is fine
- **Strict TypeScript** вЂ” `strict: true`; use type predicates (`(x): x is T =>`) instead of casts
- **GraphQL operations** live in co-located `.graphql` files; run `npm run codegen` after changing them

## GraphQL Codegen

`src/gql/` is fully generated вЂ” do not edit by hand.

- Config: `codegen.ts` (uses Shopify's public schema proxy at `https://shopify.dev/admin-graphql-direct-proxy/2026-01`)
- After editing any `.graphql` file: `npm run codegen`
- Generated exports used in source: `ExportProductsDocument`, `ExportCollectionsDocument`, `CollectionProductsDocument`, `ProductSetDocument`, `CollectionCreateDocument`, `CollectionAddProductsDocument`
- `shopifyClient.graphql(shop, SomeDocument, vars)` infers `TData` and `TVariables` from the `TypedDocumentNode` вЂ” no explicit type parameter needed at call sites

## shopifyClient

`shopifyClient.graphql(shop, document, vars?)` вЂ” accepts a `TypedDocumentNode` or a plain string (used in tests and integration helpers).

- Shop must be `config.PROD_SHOP` or `config.DEV_SHOP`; unknown shops throw immediately
- Per-shop leaky-bucket state tracks `extensions.cost.throttleStatus` from each response
- Pre-request wait when projected available < 10% of max bucket
- Retries up to 3Г— on HTTP 429 using `Retry-After` header

## Export в†’ Import Flow

1. `exportProducts` в†’ `data/products.json` (all products with variants, options, images)
2. `exportCollections` в†’ `data/collections.json` (metadata + ruleSet; manual collections include `productHandles[]`)
3. `importProducts` в†’ creates products on DEV, writes `maps/product-id-map.json`
4. `importCollections` в†’ loads map, creates collections, resolves manual membership via handleв†’GID lookup

Collections must be imported after products when manual collections are present.

## Environment

Copy `.env.example` to `.env` for normal operation:

```
PROD_SHOP=your-prod.myshopify.com
PROD_ACCESS_TOKEN=shpat_xxx
DEV_SHOP=your-dev.myshopify.com
DEV_ACCESS_TOKEN=shpat_xxx
API_VERSION=2026-01
SHOPIFY_PLAN=plus          # plus | standard  (affects bucket size: 2000 | 1000)
BATCH_SIZE=250
CONCURRENCY=10
DATA_DIR=./data
MAPS_DIR=./maps
```

## Integration Tests

Copy `.env.test.integration.example` to `.env.test.integration`. Set **both** `PROD_SHOP` and `DEV_SHOP` to the **same dev store** вЂ” the pipeline is:

```
truncate в†’ seed в†’ export в†’ truncate в†’ import в†’ verify
```

Two safety vars prevent accidental truncation of non-dev stores:

```
INTEGRATION_TEST_SHOP=your-dev.myshopify.com   # must match shop exactly
SAFE_SHOP_PATTERN=dev                           # shop domain must contain this
```

`truncateShop` throws immediately if either check fails.

## What's Not Implemented

- `idMap` вЂ” stub only; importers use `maps/product-id-map.json` directly
- Customers / Orders вЂ” deferred to Phase 2 (PII masking required)
- Bulk Operations вЂ” Phase 1 uses regular GraphQL; bulk ops for 25k+ products later
- Metafields вЂ” not included in export/import queries
